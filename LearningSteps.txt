-----------------------------Remote repo to Local------------------------------------------------
Create one repository naming git-github-learning with public and README opt.
Using VS-Code clone repository into local system using git clone link, check status also.
If file/code in local machine is modified in that case git status shows it is commited or not.

There are 4 types of status in Git
1. untracked -> New files that Git doesnt yet track
2. modified -> Changes happens in local system
3. staged -> File is ready to be commited
4. unmodified -> unchanged

                untracked          |        modified
                (New file)         |         (changed)

                    \                         /
                     \            add        /  (added but not commited)
                      \                     /
                                 staged
                                    | 
                                    | commit
                                    |
                                 unchanged   

Add & commit
````````````
add -> adds new or changed files in your working directory to the git staging area
    git add FileName (For specific file)
    git add . (For all files)

    Note :- for status from staged to unstaged 
            git restore --staged FileName

commit -> commit is the record that we have changed something in code/file.
    git commit -m "Message"
    After commit all changes we do are saved and recorded on local machine not on GitHub.

Push
````
push -> It is used to upload local repository content to remote repository.
    git push origin main 
    origin -> this is name for that repository where we had create the clone from remote.
    main -> branch (branch name)
                    push
            Local --------> Remote (Default)   

----------------------------------Local repo to Remote---------------------------------------------

Create repository inside local machine first.
Initialize the repository to git

Init Command
````````````
init -> It is used to create a new git repository on local machine (If we have project on local 
        machine then to initialze in git this command is used. Hidden .git is created inside
        local repo)
        git init 

Create repository on GitHub without description and README.md 

Link Local and Remote repository
    git remote add origin repo_link -> link both repos 

Verify which remote repository set as origin
    git remote -v   

Create README.md in local repo

Push local repository to the remote
    git push origin main -> for push/save the changes to remote(GitHub)

Workflow :- Local Git
    GitHub Repository --> Clone --> Changes --> Add --> Commit --> Push

Git Branch
```````````
Branch -> Different teams working on same project for that purpose they created 
          copy of that project and works on that specific copy. This is noting but a branch.

    git branch -> to check current branches
    git branch -r -> to check remote branches
    git branch -a -> to see all branches from remote and local machine
    git branch -M branchName -> to rename the branch 
    
Create new branch
````````````````` 
    git checkout -b branchName -> Create a new branch with branchName

Check that which is current branch. Make some changes on that new branch
After every change in any feature we have to add and commit then only changes happens.

Navigate all branches u created 
    git checkout branchName -> to navigate the branch using branchName

Delete the branch
````````````````` 
    git branch -d branchName -> to delete the existing branch from local machine only
    (While deleting any branch for that we have to checkout on that branch to another branch
    before deleting branch otherwise branch will not be deleted)

    git branch -D branchName -> to force delete

    git branch --delete branchName -> to delete the existing branch from remote also

    git fetch --prune -> remove deleted branch references

    git remote prune origin -> remove deleted branch references

    git remote remove origin -> remove remote repository
    
Check differences
``````````````````
Before merging the branch we can check the differences between branches we had created
    git diff branchName -> to compare commits,branches,files and more...
    (While checking the differences if we are on main branch and we have to check
    the differences with feature1 branch then "git diff feature1" this command shows
    differences between main and feature1 branch)

Merging branch/Code
```````````````````

We can merge the branches using two ways
1. Using Git
    git merge branchName -> to merge two branches (the branch which we are on and the branchName)


2. Using GitHub (PR)
    Pull Request (PR) -> Pull request lets you tell about changes you have pushed to a branch
                        in a repository on GitHub 
                        
    When any new branch in created then from GitHub we can merge that new branch into 
    origin (main branch). In git hub going inside that specific repository under the branch
    section -> click on new branch -> contribute -> compare and Pull Request -> confirm
    pull request using comment and discription -> sr.developer or manager checks the pull 
    request -> they accept or reject the request with comments.
